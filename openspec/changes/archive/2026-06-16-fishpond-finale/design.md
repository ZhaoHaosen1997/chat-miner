## Context

鱼塘系统已开发至 v1.16.3，当前架构：

```
services/fish_pond.py     — 核心逻辑：CRUD、指令、结算、天气、黑市
services/passive_events.py — 被动事件引擎：分组概率、DC检定、风味文本、金库
services/scheduler.py      — APScheduler：pond_events 定时触发
routers/fish_pond.py       — API 路由
models/database.py         — 数据层：fish_pond/fish_events/fish_inventory 等表
```

关键集成点：
- `settle_all_fish(group_id, date_str)` — 每日结算入口（当前：单鱼结算 + 天气应用 + 鲨鱼检定 + 黑市生成）
- `_generate_weather(group_id, date_str)` — 天气生成（当前 per-group，需改为 global）
- `trigger_passive_events(group_id)` — 被动事件主入口（精力恢复 → 分组骰事件 → 风味文本 → 金库）

设计文档 v1.16.x 原始规划中 v1.16.4~v1.16.8 包含关系网、季节、天气扩展、传奇鱼、皮肤、创意工坊等功能。本版本合并精简实现。

## Goals / Non-Goals

**Goals:**
- 鱼与鱼之间产生关系网（友谊/劲敌），让鱼塘有"社交感"
- 全服统一季节+天气，让所有群的鱼共享同一片天
- 前端 CSS 动画让季节和天气可视化
- 传说鱼有终局挑战，给养成目标
- 用户能定制鱼名、公告牌、风味文本等文本内容

**Non-Goals:**
- SVG 力导向关系图（砍掉，仅文字列表）
- 创意工坊 Lv2：自定义物种+道具（砍掉，太重）
- 鱼塘皮肤系统、数据大屏（砍掉）
- 关系类型扩展（仅友谊/劲敌两种，不包含竞争/亲密）
- 季节/天气影响被动事件概率（仅影响结算 buff，不改事件引擎）

## Decisions

### D1: 关系计算时机 — 每日结算时

**选择**：在 `settle_all_fish()` 末尾追加关系计算步骤。

**替代方案**：
- 每次事件触发时实时更新 → 会导致同一对鱼在一天内多次更新；且影响事件引擎性能
- 独立定时任务 → 增加复杂度，且与结算数据不同步

**理由**：每日结算已是现有钩子，追加步骤零成本；关系数据通常只看"最近"的，日粒度足够。

### D2: 天气全服统一 — 移除 group_id 参数

**选择**：`_generate_weather(date_str)` — 种子只依赖日期，所有群返回相同天气。

**理由**：
- 用户明确要求全群统一天气
- 简化前端展示（FishPond 不需要问"哪个群的天气"）
- 季节已是全局（基于月份），天气跟随季节才自然

**注意**：这是 **BREAKING** 变更。`settle_all_fish` 和 `get_pond_snapshot` 调用处需同步改签名。

### D3: 天气变化频率 — 每日一次

**选择**：天气每天换一次（跟随每日结算），不再更频繁变更。

**理由**：
- 用户要求"限制变化频率"
- 日粒度与季节系统（3个月粒度）形成层次感
- 结算已按天去重，天气跟随结算自然对齐

### D4: 传奇鱼任务状态存储 — 单列而非新表

**选择**：`fish_pond` 表新增 `legendary_quest_step INTEGER DEFAULT 0`。

值含义：0=未开始, 1=力量试炼待挑战, 2=智慧试炼待挑战, 3=意志试炼待挑战, 4=全部通关。

每日重置逻辑：通过 `app_settings` 记录 `legendary_quest_date_{fish_id}` 追踪上次尝试日期。

**替代方案**：建新表 `fish_legendary_quests` → 过度设计，只有 4 个状态。

**理由**：单列足够，与 fish 生命周期绑定，fish 删除时自动级联。

### D5: 创意工坊存储 — app_settings 而非新表

**选择**：3 个 `app_settings` key 存储 JSON 数组（一行一条）。

**理由**：
- 设计文档已明确此方案（v1.16.7）
- 零新表，安全（纯文本，无代码执行）
- Settings 页面现有 textarea 模式可直接复用

### D6: 前端季节/天气动画 — CSS + 绝对定位叠加层

**选择**：FishTank.vue 内用绝对定位的 `<div>` 叠加层 + CSS animation/keyframes。

- 季节：4 种粒子效果（樱花瓣/气泡/落叶/雪花），通过 CSS `@keyframes` 驱动漂浮动画
- 天气：7 种叠加层（晴天无层/雨滴/暴风雨/彩虹光晕/双彩虹/沙尘/流星）

**理由**：
- 纯 CSS 无依赖，性能好
- 不引入粒子库（如 tsParticles），保持零依赖
- 季节和天气可独立叠加（季节背景 + 天气前景）

## Risks / Trade-offs

- **[R] 关系计算首次运行可能很慢**：对历史数据首次计算需要扫描全部 fish_events。→ 首次只计算最近 30 天；后续增量只算当日。
- **[R] 天气统一后旧结算数据不一致**：历史结算记录的 weather 仍是 per-group。→ 不做历史修正，新结算从 v1.16.4 起生效。
- **[R] 传奇鱼任务全检定优势可能过强**：→ 限定"传说"阶段 + 等级≥8 才能挑战，天然限制数量（通常全群只有1-2条达标）。
- **[R] 创意工坊文本可能被滥用**：→ 纯文本存储，无代码执行；textarea 限制 5000 字符；提供"恢复默认"清空按钮。
